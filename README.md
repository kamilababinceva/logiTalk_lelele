# Как работает сам код
Этот код — это полноценный клиент для чата с графическим интерфейсом. Когда программа запускается, создаётся окно приложения через customtkinter. Внутри окна сразу создаются все основные элементы интерфейса: боковое меню, область чата, поле для ввода текста, кнопка отправки сообщения и кнопка для отправки изображений. После этого программа автоматически подстраивает размеры интерфейса под размер окна, чтобы элементы не съезжали при изменении размера приложения.

Дальше клиент пытается подключиться к серверу через сокеты по адресу localhost и порту 8080. Сокеты здесь нужны для сетевого соединения между клиентом и сервером. Если подключение прошло успешно, клиент отправляет системное сообщение о подключении пользователя, а затем запускает отдельный поток через threading. Этот поток постоянно слушает сервер и ждёт новые сообщения. Поток нужен потому, что без него окно программы зависало бы во время ожидания данных из сети.

Когда пользователь вводит сообщение и нажимает кнопку отправки, текст сначала отображается в окне чата, а потом преобразуется в специальный формат и отправляется серверу через сокет. Формат нужен, чтобы сервер понимал, что именно было отправлено: обычный текст или изображение. Сообщения разделяются символом @, поэтому сервер может определить имя пользователя и содержимое сообщения.

Если пользователь выбирает изображение, программа открывает окно выбора файла через filedialog. После выбора картинка читается как набор байтов и кодируется через base64. Это делается потому, что сокеты лучше передают текстовые данные, а не бинарные файлы напрямую. Закодированная строка отправляется серверу вместе с именем файла и именем пользователя. После отправки изображение сразу отображается в самом чате.

Когда клиент получает данные от сервера, программа собирает их в буфер и разделяет по строкам. Каждая строка передаётся в функцию обработки сообщений. Там проверяется тип сообщения. Если это обычный текст, он просто отображается в окне чата. Если это изображение, программа декодирует строку Base64 обратно в байты, создаёт из них изображение через Pillow и показывает его внутри интерфейса.

Также в программе есть боковое меню. Оно открывается и закрывается с анимацией изменения ширины. Внутри меню пользователь может изменить своё имя. После сохранения новое имя используется для всех следующих сообщений.

Вся визуальная часть построена на customtkinter, который делает интерфейс более современным по сравнению с обычным tkinter. Для изображений используется библиотека Pillow (PIL), для сети — socket, для многопоточности — threading, а base64 и io помогают передавать и восстанавливать изображения через интернет-соединение.

По сути, программа работает как мини-мессенджер: интерфейс показывает сообщения, сокеты отвечают за связь с сервером, потоки позволяют одновременно получать сообщения и не зависать окну, а Base64 позволяет передавать картинки как текст.
